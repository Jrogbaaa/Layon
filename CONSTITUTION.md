# Project Constitution — You First Gersh Influencer Insights Platform

## Mission

Help You First (Madrid, part of Gersh) grow its talent's Instagram presence by turning raw
public data into decisions the talent and the agency can act on. The product exists to
answer three questions for each influencer on the roster:

1. **Metrics** — how are they doing? (followers, engagement, growth, format performance)
2. **Creative recommendations** — what should they post next, and why? (grounded in their
   own metrics *and* current trend reports, not generic advice)
3. **Trends** — what's working in the market right now? (scraped from external trend sources)

## The Three Pillars

- **Metrics**: derived from data scraped daily from each influencer's public Instagram
  profile and recent posts (Instaloader). Stored as time-series snapshots so growth and
  trends over time are computable, not just point-in-time numbers.
- **Creative recommendations**: generated by the Gemini API once per day, using (a) the
  influencer's own metrics/post performance and (b) freshly scraped content from trend
  report sources. Recommendations must be specific to the influencer, not generic
  boilerplate — cite the metric or trend that motivated each suggestion.
- **Trend intelligence**: daily scrape of:
  - https://newengen.com/insights/instagram-trends/
  - https://www.modash.io/content-library/country/spanish-examples

## Non-Negotiables

- **Simplicity first** (Karpathy rules) — smallest implementation that satisfies the spec.
  No speculative features, no configurability that wasn't asked for.
- **Scraping etiquette** — the daily script runs once per day, with conservative delays
  between profile requests. It must never hammer Instagram or the trend sites.
- **ToS risk is acknowledged, not hidden** — scraping Instagram's web interface violates
  Meta's Terms of Service. This is a deliberate, informed choice for a small (6-profile),
  low-frequency (daily), internal tool. If this ever needs to scale or become
  ToS-compliant, the Instagram Graph API is the documented upgrade path.
- **Talent data privacy** — only public Instagram data is collected. No private account
  data, DMs, or non-public information. The platform is password-gated, not open to the
  public internet.
- **Bilingual** — recommendations and platform copy should work in Spanish and English;
  the agency and its talent operate in both.
- **No influencer-brand matching** — that capability already exists in Project-X
  (github.com/Jrogbaaa/Project-X) and is explicitly out of scope here.

## Tech Stack & Rationale

| Layer | Choice | Why |
|---|---|---|
| Scraper | Python + Instaloader | Actively maintained; instagram-php-scraper is dead since 2022 |
| Trend scraper | Python + requests/BeautifulSoup | Two known static-ish sources, no need for a browser engine |
| Scheduling | macOS `launchd` (`StartCalendarInterval`) | Runs once daily; catches up on wake if the laptop was asleep at the scheduled time — no extra automation service needed |
| Database | Supabase (Postgres) | Free hosted Postgres, no server to manage; used as a plain database (no Supabase Auth) |
| Recommendations | Google Gemini API | User-provided Google API key |
| Platform | Next.js (App Router) + Tailwind, deployed to Vercel | Matches the agency's existing Project-X deployment pattern |
| Platform auth | Single shared password (`LAYCC`) via middleware cookie gate | No per-user accounts needed; internal + talent access via one shared password |

## Definition of Done

A phase is done when it has been **run and observed to work**, not merely written:
- Scraper: manually run, rows confirmed in Supabase for all 6 handles and both trend
  sources; re-run confirmed idempotent; a broken handle confirmed to skip gracefully
  rather than crash the run.
- Scheduling: LaunchAgent loaded, confirmed to fire on schedule and catch up after wake.
- Recommendations: confirmed to reference real metrics and real trend content, not
  placeholder text.
- Platform: confirmed locally and after Vercel deploy — password gate works, dashboards
  render real scraped data.

If a step wasn't verified, say so explicitly rather than claiming it's done.
